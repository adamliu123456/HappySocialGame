# 社交对战游戏 MVP 产品方案（第一版）

## 1. 目标与定位

本项目优先验证“**社交驱动 + 可复玩 + 可赛季化**”是否成立，MVP 聚焦低同步压力的异步/回合玩法，降低首年研发和反作弊成本。

- 核心目标：
  1. 7 日留存：验证社交关系链是否提升回流。
  2. 付费转化：验证“外观 + 通行证”非数值付费的可行性。
  3. 排位健康度：验证新手公平性与高手区分度。

## 2. 玩法选型

### 2.1 MVP 主玩法（优先）

选择 **原型B：异步回合「棋盘突袭」** 作为首发：

- 单局 2–4 人，小棋盘，回合内“1 移动 + 1 攻击/技能”。
- 支持异步提交回合；若双方在线，可切换限时实时回合房。
- 新手前两局采用镜像阵容，降低理解成本并提高公平感。

### 2.2 次级拓展玩法（中期）

在核心回合玩法稳定后，引入：

- 公会周赛（借鉴原型E）
- 限时活动玩法池（轻内容扩展，不改核心结算）

## 3. 社交系统（MVP）

1. 好友系统：加好友、好友挑战、最近对手一键复战。
2. 房间系统：邀请码开房、围观席位、赛后回放分享。
3. 公会系统（轻量）：聊天、公告、每周积分榜。
4. 复盘协作：每回合可打“战术标注”，用于复盘与教学。

## 4. 排位与匹配

## 4.1 匹配策略

- 新手池：前 10 局单独匹配，限制高手炸鱼。
- 常规池：按分段 + 等待时长动态扩窗。
- 异步池：允许更宽匹配延时，提高配对成功率。

### 4.2 评分算法策略

- 1v1 默认 Elo（实现简单、迭代快）
- 2v2 / 多人局预留 TrueSkill 接口
- 可引入 Glicko RD 以改善新手和回流玩家评分置信度

## 5. 赛季与运营

- 赛季周期：4–6 周。
- 赛季目标：段位冲刺 + 每周任务 + 社交任务（组队/观战/复盘分享）。
- 赛季结算：发放限定外观、头像框、表情，不直接发数值资源。

## 6. 变现与合规

## 6.1 变现分层

1. 纯外观：角色皮肤、棋盘、击败特效、表情包。
2. 战斗通行证：任务链 + 分层奖励，增强中期目标感。
3. 限时礼包/活动：仅提供便利或外观，不售卖直接战力。

### 6.2 合规红线

- 任何付费随机奖励必须显式披露概率/规则。
- 面向未成年人地区需加强时长、消费与隐私合规。
- 跨地区发行前，分别核对平台政策与本地监管要求。

## 7. 技术路线（首年）

1. 服务端权威回合结算（避免客户端裁定）。
2. 回合日志可回放（用于观战/举报/复盘）。
3. 状态同步优先“事件流 + 快照”模型。
4. 反作弊优先做“规则校验 + 行为异常检测”两层。

## 8. 里程碑

- M1（2–4 周）：单局回合原型 + AI 机器人 + Elo。
- M2（4–8 周）：异步对局、好友挑战、回放。
- M3（8–12 周）：赛季系统、通行证、基础公会周赛。
